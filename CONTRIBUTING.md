# Contributing to Attentube

Thank you for your interest in contributing. Attentube is intentionally simple — contributions should keep it that way.

## Principles

- **Minimal** — Every feature must earn its place. If it adds distraction, it doesn't belong.
- **No dependencies** — Pure HTML/CSS/JS. No frameworks, no bundlers, no npm dependencies in the main app.
- **Clean code** — Well-commented, readable, consistent style.

## How to Contribute

1. **Fork** the repository
2. **Create a branch**: `git checkout -b feature/your-feature`
3. **Make your changes** — follow the existing code style
4. **Test locally**: open `index.html` in a browser (with a local server)
5. **Commit**: use conventional commits — `feat:`, `fix:`, `docs:`, `style:`, `refactor:`
6. **Open a Pull Request** with a clear description

## What We Welcome

- Bug fixes
- Accessibility improvements
- Performance improvements
- New features from the [roadmap](README.md#roadmap)
- Documentation improvements

## What We Avoid

- Adding npm dependencies to the core app
- Recommendation features
- Autoplay queues
- Anything that resembles YouTube's engagement-maximizing UI

## Code Style

- 2-space indentation
- ES modules (`import`/`export`) throughout
- JSDoc comments on all exported functions
- CSS custom properties for all values (no magic numbers)

## Questions?

Open an issue — we're happy to discuss ideas before you start building.
